# Plan 20 — Statewide Expansion (Phase 3)

**Status:** Draft — deferred  
**Priority:** Future  
**Depends on:** Plan 19 (recommended)  
**Estimated effort:** Very large (ongoing)

---

## Goal

Expand CourtFlow AI from **NYC 5 counties** to all **62 New York State counties** with county-specific forms and rules.

## Requirements reference

- PRD Ch. 39 — Phase 3 Scope
- Business idea doc — county-specific form system across NY
- MVP roadmap — statewide expansion

## Prerequisites

- MVP + Phase 2 stable in NYC
- Admin tools for county rules (Plan 14)
- Forms catalog pipeline proven (Plan 05)
- Verification system operational (Plan 17)

## Scope (high level)

### In scope

- County data model extension beyond NYC
- Forms catalog ingestion per county
- Workflow variants or county conditions on shared workflows
- County selector UX (not NYC-default)
- Regional court routing (Supreme Court counties outside NYC)
- Phased rollout plan by region

### Out of scope

- Other states
- Federal courts

## Deliverables

1. Statewide expansion strategy (county priority tiers)
2. Data pipeline for non-NYC form imports
3. Per-county QA process
4. Pricing/ops model for maintaining 62 counties

## Acceptance criteria

- TBD per rollout wave (e.g. “Wave 1: Downstate counties”)

## Risks

- Form proliferation — 62× court combinations
- Maintenance burden without admin tooling and verification automation
